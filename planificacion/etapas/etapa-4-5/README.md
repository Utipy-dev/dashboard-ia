# Etapas 4 y 5 — Extensión completa + App nativa + Transcriptor

**Estado:** 💭 plan (horizonte lejano)
**Depende de:** Etapa 3 completa

## Nota sobre el nivel de detalle

Los bloques de etapas 4 y 5 están escritos a nivel de intención, no de implementación. Las decisiones técnicas definitivas se tomarán cuando etapa 3 esté funcionando. Lo que no cambia es el **por qué**: qué problema resuelve cada bloque y qué valor aporta al usuario.

---

## Etapa 4 — La extensión como producto

**Pitch:** En etapas 1-3, la extensión de Chrome es auxiliar: captura texto, inyecta mensajes. En etapa 4, se convierte en una superficie completa. El usuario puede usar el dashboard entero sin salir del navegador.

**Bloques:**
- 4.0: UI completa en la extensión (no solo el popup de 20x20)
- 4.1: Dashboard completo accesible desde el panel lateral de Chrome
- 4.2: Firma de código y publicación definitiva en Chrome Web Store

---

## Etapa 5 — App nativa + Transcriptor propio

**Pitch:** El servidor local tiene que arrancar cuando abre el PC. El dashboard tiene que estar en el dock/menú inicio. Y el transcriptor tiene que ser mejor que el de Claude: mezcla voz y escritura, funciona offline, guarda historial propio.

**Bloques:**
- 5.0: Shell de app nativa que envuelve el servidor local
- 5.1: Transcriptor con mezcla voz + escritura
- 5.2: Release final y guía completa publicada
